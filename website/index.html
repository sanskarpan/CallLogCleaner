// process

Four steps to
complete control.

No setup required. No account. Plug in your iPhone, open the app, and you're done in minutes.

01
Select Backup
The app automatically finds all iPhone backups stored on your Mac. Select one — encrypted backups are required.
02
Enter Password
Provide the backup password you set in Finder. The app derives the decryption key — nothing leaves your machine.
03
Select Records
Filter by contact, date, call type, or direction. Use Smart Select to find missed calls, duplicates, or short calls instantly.
04
Delete & Restore
Records are deleted from the backup file, re-encrypted, and written back. Restore via Finder to apply to your iPhone.
// features

Built for
power users.

🔓
Full Backup Decryption
Parses Apple's BackupKeyBag TLV format, derives the KEK via two-stage PBKDF2, unwraps class keys, and decrypts both the manifest and individual files.
AES-256-CBC RFC 3394 PBKDF2 CommonCrypto
🔍
Powerful Filtering
Filter by contact name or number, date range, call type (Phone, FaceTime Video, FaceTime Audio), direction, and answered/missed status. Filters stack.
Smart Select Context Menus Multi-sort Export CSV/JSON
🗑
Surgical Deletion
Deletes records from ZCALLRECORD and cleans orphaned handle references. Re-encrypts the database and re-packs it into the backup with an updated Manifest.
SQLite3 Re-encryption Manifest Update Zero Data Loss

// under the hood

Every detail
considered.

Encryption chain
// 2-stage PBKDF2 key derivation
PBKDF2(SHA256, password, DPSL, dpic)
  → PBKDF2(SHA1, stage1, SALT, iter)
  → KEK (key-encryption key)

// RFC 3394 key unwrap
CCSymmetricKeyUnwrap(kCCWRAPAES, KEK, WPKY)
  → classKey

// File decryption
AES-256-CBC(fileKey, iv=0x00×16)
  → CallHistory.storedata
What gets cleaned up
  • ZCALLRECORD rows matching selected Z_PK values
  • Z_2REMOTEPARTICIPANTHANDLES junction table entries
  • Orphaned ZHANDLE rows with no remaining call references
  • Manifest.db fileID updated after re-encryption
  • Old backup file removed; new hash location written
